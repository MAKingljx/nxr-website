## Admin Access Memory

- Public `https://nxrgrading.com/admin` is not the admin entrance. It is intentionally configured to return `404`.
- The real public admin entry URL is `https://nxrgrading.com/x7k9m2q4r8v6c3p1`.
- Local/admin backend route checks should use `http://127.0.0.1:8081/admin/login` on the server side, not `127.0.0.1:8080/admin/login`.
- Future smoke tests and debugging should treat `/admin` as a blocked decoy path and the hidden route as the real public admin path.

## Backup Retention Memory

- Keep at most 2 local backup snapshots for this project.
- Default rule: preserve the 2 newest entries in `local_backups/` and remove older ones during cleanup.

## Database Authorization Memory

- Strong standing rule: without explicit user authorization in the current session, do not sync, replace, restore, or overwrite any database under `Data/`.
- Do not infer permission for later database writes from an earlier approval. Each database-changing action requires a fresh explicit user instruction.
- Normal code sync must keep `Data/` excluded.

## Remote Restart Memory

- The production main site and admin backend are separate Python processes.
- Main site: `/usr/bin/python3 app.py` on `127.0.0.1:8080`.
- Admin backend: `/usr/bin/python3 /root/nxr_website/nxr_admin/app_updated.py` on `127.0.0.1:8081`.
- Admin template/code changes require an explicit restart of `nxr_admin/app_updated.py`; restarting only `app.py` is not sufficient.
- Preferred helper: `scripts/restart_remote_apps.sh`.
