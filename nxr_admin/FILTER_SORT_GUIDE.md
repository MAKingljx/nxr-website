# 已批准条目筛选和排序功能指南

## 功能概述
在已批准条目页面（`/admin/entries?status=approved`）新增了强大的筛选和排序功能，方便用户快速查找和管理已批准的卡片数据。

## 访问方式
1. 登录管理后台：`http://localhost:8081/admin`
2. 点击左侧菜单的"Entries"
3. 点击"Approved"标签或直接访问：`http://localhost:8081/admin/entries?status=approved`

## 筛选功能

### 1. 卡片名称筛选（模糊查询）
- **功能**：支持模糊查询卡片名称
- **使用**：在"Card Name"输入框中输入关键词
- **示例**：输入"Charizard"会匹配所有包含"Charizard"的卡片名称
- **特点**：不区分大小写，支持部分匹配

### 2. 最终评级筛选
- **功能**：按最终评级筛选
- **使用**：从"Final Grade"下拉列表中选择评级
- **选项**：显示数据库中所有存在的评级（如8, 8.5, 9, 9.5, 10, Pristine 10）
- **默认**："All Grades"（显示所有评级）

### 3. 系列名称筛选
- **功能**：按卡片系列筛选
- **使用**：从"Set Name"下拉列表中选择系列
- **选项**：显示数据库中所有存在的系列名称
- **默认**："All Sets"（显示所有系列）

### 4. 语言筛选
- **功能**：按语言代码筛选
- **使用**：从"Language"下拉列表中选择语言
- **选项**：EN, JP, CT, CS, IN, KO, TH, Other
- **默认**："All Languages"（显示所有语言）

## 排序功能

### 1. 排序字段选择
- **Approval Date**：按批准日期排序（默认）
- **Card Name**：按卡片名称字母顺序排序
- **Final Grade**：按最终评级排序
- **Set Name**：按系列名称排序
- **Language**：按语言代码排序

### 2. 排序顺序选择
- **Descending**：降序排列（默认）
- **Ascending**：升序排列

### 3. 表格表头点击排序
在已批准页面，表格的表头支持直接点击排序：
- **Card Name**：点击卡片名称表头排序
- **Final Grade**：点击最终评级表头排序
- **Approval Date**：点击批准日期表头排序

点击表头会在当前筛选条件下切换排序方式和顺序。

## 使用示例

### 示例1：查找特定卡片
```
需求：查找所有包含"Pikachu"的已批准卡片，按卡片名称升序排列

操作：
1. 在"Card Name"输入框输入：Pikachu
2. 在"Sort By"选择：Card Name
3. 在"Order"选择：Ascending
4. 点击"Apply Filters"
```

### 示例2：查看特定评级的卡片
```
需求：查看所有评级为9.5的卡片，按批准日期降序排列

操作：
1. 在"Final Grade"选择：9.5
2. 在"Sort By"选择：Approval Date
3. 在"Order"选择：Descending
4. 点击"Apply Filters"
```

### 示例3：多条件筛选
```
需求：查看Pokemon品牌、英语(EN)、评级9以上的卡片

操作：
```

## 功能特点

### 1. 智能筛选
- 筛选条件会保留在URL中，方便分享和书签
- 筛选表单会记住当前的选择
- 支持组合多个筛选条件

### 2. 实时反馈
- 显示找到的条目数量
- 表头显示当前排序状态
- 筛选条件清晰可见

### 3. 用户友好
- 清晰的表单标签和说明
- 合理的默认值
- 一键清除所有筛选

### 4. 性能优化
- 数据库查询优化，避免全表扫描
- 下拉选项动态生成，避免硬编码
- 支持模糊查询的索引优化

## 技术实现

### 后端逻辑
```python
# 构建动态查询
query = "SELECT * FROM temp_cards WHERE status = 'approved'"
params = []

if card_name_filter:
    query += " AND card_name LIKE ?"
    params.append(f"%{card_name_filter}%")

if final_grade_filter:
    query += " AND final_grade_text = ?"
    params.append(final_grade_filter)

# 添加排序
query += f" ORDER BY {sort_by} {sort_order.upper()}"
```

### 前端实现
- 使用GET方法传递筛选参数
- 表头点击生成新的排序URL
- 表单自动填充当前筛选值
- 响应式设计，适配各种屏幕

## 最佳实践

## 常见问题

### Q1：为什么某些筛选选项为空？
A：下拉选项从数据库中动态生成，如果数据库中没有相应数据，选项会为空。

### Q2：模糊查询支持哪些通配符？
A：使用SQL的LIKE语法，`%`表示任意字符，`_`表示单个字符。

### Q3：如何保存当前的筛选条件？
A：筛选条件保存在URL中，可以将URL加入书签或分享给他人。

### Q4：筛选会影响其他页面吗？
A：不会，筛选只影响当前页面，其他页面保持默认状态。
